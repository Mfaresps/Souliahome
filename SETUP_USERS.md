# 🚀 كيفية تطبيق المستخدمين الجدد

## ⚡ الطريقة الأسرع:

### 1. توقف التطبيق الحالي (Ctrl+C في كل terminal)
```
إذا كنت تشغل Backend أو Frontend، توقفهما أولاً
```

### 2. حذف قاعدة البيانات
اختر **واحدة من هذه الطرق**:

#### الطريقة أ: إذا كنت تستخدم MongoDB Atlas (سحابي)
```bash
1. افتح MongoDB Atlas Dashboard
2. اذهب إلى Database Collections
3. احذف database باسم "soulia"
4. سيتم إعادة إنشاؤه تلقائياً مع البيانات الجديدة
```

#### الطريقة ب: إذا كنت تستخدم MongoDB local
```bash
# افتح MongoDB Compass أو terminal جديد:
mongosh  # أو mongo في الإصدارات الأقدم

# ثم شغل:
> use soulia
> db.dropDatabase()
> exit
```

#### الطريقة ج: حذف ملف قاعدة البيانات (إن وجد)
```bash
# على Windows
rmdir /s "C:\data\db\soulia"

# على Mac/Linux
rm -rf /data/db/soulia
```

### 3. شغّل Backend مرة أخرى
```bash
cd backend
npm run start
```

✅ البذر سيعمل تلقائياً (موجود في `onModuleInit()`)
✅ سترى في الـ console: "Seeded user: admin", "Seeded user: admin2", إلخ

### 4. جرّب تسجيل الدخول
```
اسم المستخدم: admin
كلمة المرور: Fares@2024
```

---

## 📋 جدول الملخص:

### ✅ ما تم تغييره:
- ✏️ اسم المدير الرئيسي: "المدير" → **"Fares"**
- ➕ مدير ثاني جديد: **محمد الإدارة** (admin2)
- ➕ 5 موظفي مبيعات جدد مع صلاحياتهم
- ➕ 5 محاسبين جدد مع صلاحياتهم
- 🗑️ تم حذف المستخدم الوهمي القديم (staff)

### 🔐 الصلاحيات تلقائياً:
- **المديرين (Admins)**: جميع الصلاحيات ✅
- **موظفو المبيعات**: مبيعات، مخزون، عملاء، منتجات
- **المحاسبين**: تقارير، خزنة، مصاريف، موردين

---

## ❓ أسئلة شائعة:

**س: هل يمكن تغيير كلمات المرور لاحقاً؟**
ج: نعم! من لوحة "إدارة المستخدمين" (admin فقط)

**س: هل قاعدة البيانات محمية؟**
ج: **نعم** - تحتاج إلى تسجيل دخول + token JWT

**س: ماذا لو نسيت كلمة مرور Fares؟**
ج: حذّف المستخدم وأعد التشغيل (سيتم إعادة بناء قاعدة البيانات)

**س: كيف أضيف مستخدمين جدد من الواجهة؟**
ج: سجل دخول كـ admin → اذهب لـ "المستخدمين" → "إضافة مستخدم جديد"
