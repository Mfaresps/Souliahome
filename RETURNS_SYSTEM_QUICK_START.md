# نظام الاسترجاع والاستبدال - دليل البدء السريع
## Returns & Exchanges - Quick Start Guide

---

## 🚀 البدء (Getting Started)

### الخطوة 1: الوصول للنظام
```
الموقع: https://your-domain.com
القسم: المستخدمون → الاسترجاعات والاستبدالات
أو: Dashboard → Returns & Exchanges
```

### الخطوة 2: فهم الفرق
```
🔄 استرجاع (Return)
   العميل يرجع المنتج كاملاً
   ← الشركة تردّ الفلوس

🔄 استبدال (Exchange)
   العميل يرجع منتج A
   ← يأخذ منتج B بدلاً منه
```

## 📝 طلب جديد

### إنشاء طلب استرجاع بسيط

**الخطوات:**
```
1. اضغط: "+ إضافة طلب استرجاع"

2. ملأ البيانات:
   ┌─────────────────────────────┐
   │ رقم الفاتورة: INV-2025-0001  │
   │ العميل: أحمد محمد            │
   │ الهاتف: 01001234567          │
   └─────────────────────────────┘

3. اختر الأصناف المرتجعة:
   ☑ تي شيرت أحمر × 1 (150 ج)
   ☐ جوارب سوداء × 2 (50 ج)

4. السبب:
   [🔽 اختر السبب]
   ├ تلف الشحنة
   ├ شحنة خاطئة
   ├ سبب آخر
   
5. ملاحظات (اختياري):
   "الملابس وصلت ممزقة"

6. طريقة الرد:
   [🔽 كاش / فودافون / Instapay]

7. اضغط: "✅ تأكيد الطلب"
```

**النتيجة:**
```
✅ طلب تم إنشاؤه بنجاح
   ID: RET-2025-000001
   الحالة: معلق (قيد المراجعة)
   
   ⏳ الخطوة التالية:
   سيراجعه الإداري خلال 24 ساعة
```

### إنشاء طلب استبدال

**الخطوات:**
```
1. اضغط: "+ طلب استبدال جديد"

2. رقم الفاتورة: INV-2025-0001

3. الأصناف المرتجعة:
   ☑ جوال Samsung A12 × 1 (3000 ج)

4. الأصناف البديلة:
   ☑ جوال Samsung A13 × 1 (3500 ج)

5. مؤشر السعر:
   ✅ يظهر تلقائياً:
   
   المرتجع: 3000 ج
   البديل:  3500 ج
   الفرق:   500 ج ← ❌ العميل يدفع إضافي

6. السبب:
   [رغبة العميل بصنف آخر]

7. اضغط: "✅ إرسال الطلب"
```

**النتيجة:**
```
✅ طلب استبدال مقبول
   ID: EXC-2025-000001
   الحالة: معلق
   
   ⚠️ ملاحظة:
   المبلغ المستحق: 500 ج
   سيتم إرسال فاتورة دين للعميل
```

## ⚠️ رسائل الخطأ الشائعة

### عند الإنشاء

| الخطأ | السبب | الحل |
|------|------|-----|
| "فترة الاسترجاع انتهت" | الفاتورة قديمة أكثر من 14 يوم | تواصل مع الإدارة |
| "الصنف غير موجود" | الصنف ليس من الفاتورة | تحقق من رقم الصنف |
| "تم استرجاع هذا سابقاً" | نفس الصنف استُرجع من قبل | تواصل بطلب جديد |

### عند الموافقة (إداري)

| الخطأ | السبب | الحل |
|------|------|-----|
| "الخزنة غير كافية" | لا يوجد رصيد كافٍ | أضف رصيد أولاً |
| "الصنف غير متاح" | الاستبدال غير موجود | اختر بديل آخر |
| "بيانات ناقصة" | هاتف أو عنوان ناقص | أكمل البيانات |

## 🎯 أفضل الممارسات (Best Practices)

### للعميل:
```
✅ افعل:
   • أرسل صورة للعيب
   • اذكر رقم الفاتورة بدقة
   • رد الصنف في حالة جيدة

❌ لا تفعل:
   • لا تنتظر أكثر من 14 يوم
   • لا ترسل صنف مختلف
   • لا تطلب استرجاع غير مُوثق
```

### للإداري:
```
✅ افعل:
   • راجع الطلب خلال 24 ساعة
   • تحقق من الخزنة قبل الموافقة
   • أرسل تأكيد فوري للعميل

❌ لا تفعل:
   • لا توافق على طلب به أصناف غير متطابقة
   • لا تتجاهل الفحوصات الأمنية
   • لا تسجل معاملات يدوية (دع النظام يفعل)
```
