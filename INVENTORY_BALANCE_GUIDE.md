# نظام حساب أرصدة المخزن الحي
## Real-Time Inventory Balance System

---

## المعادلة الأساسية

```
الرصيد الحالي = الرصيد الافتتاحي + المشتريات + إرجاع للمخزن - المبيعات
```

### Components:
- **الرصيد الافتتاحي** (Opening Balance): الكمية الأولية عند بداية الفترة
- **المشتريات** (Purchases): الكميات المشتراة من الموردين
- **إرجاع للمخزن** (Returns to Stock): الكميات المعادة من العملاء (مرتجعات/استبدالات)
- **المبيعات** (Sales): الكميات المباعة للعملاء
- **الرصيد الحالي** (Current Balance): الكمية المتبقية في المخزن

---

## أمثلة عملية

### مثال 1: حساب بسيط

**الصنف:** قميص أحمر

| الحقل | القيمة |
|------|--------|
| الرصيد الافتتاحي | 100 وحدة |
| المشتريات | 50 وحدة |
| إرجاع للمخزن | 5 وحدات |
| المبيعات | 80 وحدة |

**الحساب:**
```
الرصيد الحالي = 100 + 50 + 5 - 80 = 75 وحدة
```

**الحالة:** متوفر ✅

---

### مثال 2: مخزون منخفض

**الصنف:** حذاء رياضي

| الحقل | القيمة |
|------|--------|
| الرصيد الافتتاحي | 30 وحدة |
| المشتريات | 20 وحدة |
| إرجاع للمخزن | 2 وحدة |
| المبيعات | 45 وحدة |
| الحد الأدنى | 15 وحدة |

**الحساب:**
```
الرصيد الحالي = 30 + 20 + 2 - 45 = 7 وحدات
```

**الحالة:** منخفض ⚠️ (أقل من 15 وحدة)

---

### مثال 3: مخزون نفد

**الصنف:** شنطة جلدية

| الحقل | القيمة |
|------|--------|
| الرصيد الافتتاحي | 10 وحدات |
| المشتريات | 8 وحدات |
| إرجاع للمخزن | 0 |
| المبيعات | 20 وحدة |

**الحساب:**
```
الرصيد الحالي = 10 + 8 + 0 - 20 = -2 (يُعتبر 0)
```

**الحالة:** نفد 🚫 (المخزن انتهى)

---

## تقسيمات الحالة

### 1. متوفر ✅ (Available)
- **الشرط:** الرصيد الحالي > الحد الأدنى
- **اللون:** أخضر
- **الإجراء:** يمكن البيع بحرية

### 2. منخفض ⚠️ (Low Stock)
- **الشرط:** 0 < الرصيد الحالي ≤ الحد الأدنى
- **اللون:** برتقالي
- **الإجراء:** تنبيه المشتريات بالشراء

### 3. نفد 🚫 (Out of Stock)
- **الشرط:** الرصيد الحالي ≤ 0
- **اللون:** أحمر
- **الإجراء:** منع البيع، شراء طارئ

---

## حركات الإدخال والإخراج

### الإدخالات (Inbound) ➕
```
الإدخالات = المشتريات + إرجاع للمخزن
```

**أمثلة:**
- شراء من المورد: +50 وحدة
- استرجاع من عميل: +5 وحدات
- استبدال (الأصنف الجديدة): +10 وحدات

### الإخراجات (Outbound) ➖
```
الإخراجات = المبيعات
```

**أمثلة:**
- بيع للعميل: -80 وحدة
- مرتجع عميل (الأصناف القديمة): -5 وحدات

### صافي الحركة (Net) 📊
```
صافي الحركة = الإدخالات - الإخراجات
```

---

## حالات خاصة

### 1. التحديث الفوري
عند إضافة أو تعديل حركة:
```
الرصيد_الجديد = الرصيد_الحالي + التأثير_الجديد
```

**مثال:** بيع 10 وحدات من رصيد 75
```
75 - 10 = 65 وحدة
```

### 2. استبدال (Exchange)
عند استبدال أصناف:
```
الإخراج: - كمية الأصناف القديمة (من رصيد A)
الإدخال: + كمية الأصناف الجديدة (لرصيد B)
```

**مثال:** استبدال 5 قمصان حمراء بـ 5 قمصان زرقاء
```
قميص أحمر: الرصيد - 5
قميص أزرق: الرصيد + 5
```

### 3. الإرجاع مع تصحيح الأسعار
لا يؤثر على الكمية:
```
الرصيد الحالي = الرصيد الحالي  (دون تغيير)
(الفرق السعري يُعالج في الخزنة، ليس في المخزن)
```

---

## نسبة المخزون

```
نسبة المخزون (%) = (الرصيد الحالي ÷ الحد الأدنى) × 100
```

### معنى النسبة:
- **100%**: الرصيد = الحد الأدنى تماماً
- **200%**: الرصيد ضعف الحد الأدنى
- **50%**: الرصيد نصف الحد الأدنى
- **0%**: المخزن فارغ تماماً

---

## الرسم البياني لتتبع الرصيد

```
الرصيد الافتتاحي
      |
      | + المشتريات ➕
      | + إرجاع للمخزن ➕
      | - المبيعات ➖
      |
      ↓
الرصيد الحالي
      |
      ├─→ متوفر ✅ (> الحد الأدنى)
      ├─→ منخفض ⚠️ (≤ الحد الأدنى)
      └─→ نفد 🚫 (≤ صفر)
```

## التحديثات المدعومة

### من حركات البيع (Sales)
```typescript
// عند إنشاء حركة بيع
الرصيد -= الكمية_المباعة
```

### من حركات الشراء (Purchases)
```typescript
// عند استلام طلبية من المورد
الرصيد += الكمية_المشتراة
```

### من المرتجعات (Returns)
```typescript
// عند إرجاع العميل للأصناف
الرصيد += الكمية_المعادة
```

### من الاستبدالات (Exchanges)
```typescript
// النوع A (المرتجع)
النوع_A_الرصيد -= الكمية_المرتجعة

// النوع B (البديل)
النوع_B_الرصيد += الكمية_البديلة
```

## قواعد التحقق

### 1. يجب توفر المخزون للبيع
```
if (الرصيد_الحالي >= الكمية_المطلوبة) ✅
else ❌ "المخزون غير كافي"
```

### 2. لا توجد كميات سالبة
```
الرصيد_الحالي = max(0, الحساب)
```

### 3. التحديث فوري
```
عند كل حركة → تحديث فوري للرصيد
→ تحديث الحالة (متوفر/منخفض/نفد)
→ عرض في لوحة التحكم
```

## الخلاصة

النظام يحسب الرصيد الحالي بطريقة **حية وفورية** معتمداً على:
- **الرصيد الافتتاحي** كنقطة انطلاق
- **المشتريات والإرجاعات** كإضافات
- **المبيعات** كخصومات
- **النتيجة النهائية** تحدد توفر الصنف

جميع الحسابات تتم **تلقائياً وفورياً** مع كل حركة جديدة في النظام.
